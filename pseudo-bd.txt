Se requiere una base de datos en PostgreSQL para llevar un control de rutinas de ejercicios en casa y gimnasio. Al mismo tiempo, se espera poder crear
reportes periodicos (semanal/quincenal/mensual/trimestral/semestral/anual) de cada usuario esto se hará con un resultado en Excel, si es posible con gráficos.
Paso seguido, se usará POO con Python para crear la interfaz gráfica. Las opciones están con Tkinter o una combinación de HTML, CS5 y Javascript.

Para ello, se requiere:

- Una tabla de usuarios y sus datos personales.
- Una tabla de ejercicios disponibles para poder hacer tanto en casa como en gimnasio
- Una tabla de grupos musculares, para poder identificar qué parte del cuerpo se trabajó
- Una tabla de los músculos que trabajará cada ejercicio
- Una tabla de rutinas recomendadas de manera predefinida si no se quiere crear una (es necesario crear opciones de niveles: principiante, intermedio, avanzado)
- Una tabla que soluciona un problema que puede surgir como el de asignar muchos músculos a muchos ejercicios, es decir, una relación muchos a muchos es necesaria una tabla puente


CREAR TABLA usuarios:
    id_usuario: entero auto-incremental, clave primaria
    nombre_usuario: texto máximo 50 caracteres, obligatorio
    apellido_usuario: texto máximo 50 caracteres, obligatorio
    email_usuario: texto máximo 100 caracteres, obligatorio, único
    edad_usuario: entero entre 14 y 80, obligatorio
    estatura_usuario: entero entre 130 y 250 cm, obligatorio
    peso_usuario: decimal(5,2) entre 30.00 y 200.00 kg, obligatorio
    genero_usuario: texto máximo 10 caracteres, valores permitidos: 'masculino', 'femenino', 'otro'

CREAR TABLA grupos_musculares:
    id_grupo: entero pequeño auto-incremental, clave primaria
    nombre_grupo: texto máximo 100 caracteres, obligatorio, único

CREAR TABLA musculos:
    id_musculo: entero pequeño auto-incremental, clave primaria
    nombre_musculo: texto máximo 100 caracteres, obligatorio
    id_grupo_muscular: entero pequeño, clave foránea hacia grupos_musculares(id_grupo)

CREAR TABLA ejercicios:
    id_ejercicio: entero pequeño auto-incremental, clave primaria
    nombre_ejercicio: texto máximo 100 caracteres, obligatorio, único
    descripcion_ejercicio: texto máximo 500 caracteres, obligatorio
    tipo_ejercicio: texto máximo 50 caracteres, valores permitidos: 'casa', 'gimnasio', 'ambos'


- La siguiente tabla es una tabla puente, ya que el proyecto requiere saber qué músculos trabaja cada ejercicio. 
- Esto genera una relación muchos a muchos; por lo tanto, no sería posible incluir más de un músculo por ejercicio directamente en las tablas de ejercicios o músculos sin romper la normalización.
- Además, las bases de datos relacionales no permiten relaciones muchos a muchos de forma directa entre dos tablas.

CREAR TABLA ejercicios_musculos:
    id_ejercicio: entero pequeño, clave foránea hacia ejercicios(id_ejercicio)
    id_musculo: entero pequeño, clave foránea hacia musculos(id_musculo)
    importancia: texto máximo 20 caracteres, valores permitidos: 'principal', 'secundario', 'estabilizador'
    CLAVE PRIMARIA COMPUESTA: (id_ejercicio, id_musculo)
    ELIMINAR EN CASCADA al eliminar ejercicio o músculo

CREAR TABLA rutinas:
    id_rutina: entero auto-incremental, clave primaria
    nombre_rutina: texto máximo 100 caracteres, obligatorio, único
    descripcion: texto máximo 500 caracteres, obligatorio
    objetivo: texto máximo 30 caracteres, valores permitidos: 'fuerza', 'hipertrofia', 'resistencia'
    nivel: texto máximo 20 caracteres, valores permitidos: 'principiante', 'intermedio', 'avanzado'
    creado_en: timestamp, valor por defecto: fecha/hora actual

CREAR TABLA ejercicios_en_rutina:
    id_rutina: entero, clave foránea hacia rutinas(id_rutina)
    id_ejercicio: entero pequeño, clave foránea hacia ejercicios(id_ejercicio)
    series: entero pequeño mayor a 0, obligatorio
    repeticiones: entero pequeño mayor a 0, obligatorio
    descanso_seg: entero pequeño mayor o igual a 0, opcional
    orden_ejercicio: entero pequeño mayor a 0, opcional
    intensidad: texto máximo 100 caracteres, valores permitidos: 'baja', 'media', 'alta'
    CLAVE PRIMARIA COMPUESTA: (id_rutina, id_ejercicio)
    ELIMINAR EN CASCADA al eliminar rutina

### POSIBLES tablas

CREAR TABLA sesiones_entrenamiento:
    id_sesion: entero auto-incremental, clave primaria
    id_usuario: entero, clave foránea hacia usuarios
    id_rutina: entero, clave foránea hacia rutinas
    fecha_sesion: fecha, obligatorio
    duracion_minutos: entero pequeño
    notas: texto máximo 500 caracteres

CREAR TABLA progreso_usuario:
    id_progreso: entero auto-incremental, clave primaria
    id_usuario: entero, clave foránea hacia usuarios
    fecha_medicion: fecha, obligatorio
    peso_actual: decimal(5,2)
    medida_pecho: decimal(4,1)
    medida_cintura: decimal(4,1)
    medida_brazo: decimal(4,1)
    notas: texto máximo 300 caracteres

CREAR TABLA rendimiento_ejercicio:
    id_rendimiento: entero auto-incremental, clave primaria
    id_sesion: entero, clave foránea hacia sesiones_entrenamiento(id_sesion), obligatorio
    id_ejercicio: entero pequeño, clave foránea hacia ejercicios(id_ejercicio), obligatorio
    serie_numero: entero pequeño mayor a 0, obligatorio
    repeticiones_realizadas: entero pequeño mayor o igual a 0, obligatorio
    peso_utilizado: decimal(5,2) mayor o igual a 0, opcional
    tiempo_descanso_seg: entero pequeño mayor o igual a 0, opcional
    dificultad_percibida: entero pequeño entre 1 y 10, opcional (escala RPE)
    notas: texto máximo 200 caracteres, opcional
    
    RELACIONES:
        - ELIMINAR EN CASCADA al eliminar sesión de entrenamiento
        - RESTRINGIR eliminación si se referencia ejercicio

CREAR TABLA objetivos_usuario:
    id_objetivo: entero auto-incremental, clave primaria
    id_usuario: entero, clave foránea hacia usuarios(id_usuario), obligatorio
    tipo_objetivo: texto máximo 30 caracteres, obligatorio
        valores permitidos: 'peso', 'masa_muscular', 'fuerza', 'resistencia', 'medida_corporal'
    descripcion: texto máximo 200 caracteres, obligatorio
    valor_objetivo: decimal(6,2), opcional (valor numérico del objetivo)
    unidad_medida: texto máximo 20 caracteres, opcional (kg, cm, minutos, repeticiones, etc.)
    fecha_inicio: fecha, obligatorio, valor por defecto: fecha actual
    fecha_limite: fecha, opcional
    estado: texto máximo 20 caracteres, obligatorio, valor por defecto: 'activo'
        valores permitidos: 'activo', 'completado', 'pausado', 'cancelado'
    fecha_completado: fecha, opcional
    
    RESTRICCIONES:
        - fecha_limite debe ser posterior a fecha_inicio (si se especifica)
        - ELIMINAR EN CASCADA al eliminar usuario
